[APM] Ardupilot 펌웨어 업그레이드

Erle-Jill의 컴파스 방향이 계속 제대로 잡히지 않는 문제가 있어,
Ardupilot 펌웨어를 다시 깔아보기로 함.

Ardupilot Firmware upgrade:
다음을 따라 진행

참고페이지
윗 페이지 내용 중,
 
1. 빌드된 바이너리 파일을 받아 로딩하는 방법으로 진행

다음 경로의 최신 펌웨어 파일을 로컬 컴퓨터에 다운로드
Erle-Copter: http://firmware.ardupilot.org/Copter/latest/erlebrain2-quad/

ssh erle@10.0.0.1  Erle 접속
    mv ArduCopter.elf ArduCopter.elf.old 이전 펌웨어 이름 변경(ArduCopter.elf -> ArduCopter.elf.old)
    sudo systemctl stop apm.service apm 서비스 정지
    exit Erle 접속 해제

scp ~/ArduCopter.elf erle@10.0.0.1:~/ArduCopter.elf 펌웨어 파일 다운로드 경로(이 경우는 로컬컴퓨터의 홈 디렉토리)에서 Erle로 바이너리 파일 전송
ssh erle@10.0.0.1 Erle 접속 # launch manually binary file Ctrl+C to close
    (chmod chmod 755 ArduCopter.elf) 필요시 펌웨어의 실행 권한 부여
    sudo ./ArduCopter.elf 전송한 펌웨어 파일 실행
    # start apm.service sudo systemctl start apm.service apm 서비스 시작
    exit Erle 접속 해제

펌웨어 업그레이드 이후 컴파스 방향 문제:
APM Planner에 연결해 확인한 결과 컴파스 방향 제대로 잡힘
APM Planner 설정 등을 다시 잡아줄 필요 없이 기존 설정으로 바로 접속되며, 정상 작동함


2. 소스 받아 직접 빌드하는 방법

동일한 참고페이지  (Build APM using make)

1에서와 마찬가지로 ArduCopter.elf를 백업해 둠
mv ArduCopter.elf ArduCopter.elf.old

소스 받아 빌드
sudo apt-get install git  
git clone http://github.com/diydrones/ardupilot    
ardupilot/Tools/scripts/install-prereqs-ubuntu.sh -y
cd ardupilot
cd ArduCopter
make clean
make erlebrain2 -j4   

     cf. 만약 빌드 중 locale 관련 에러메시지가 뜨면(ko_KR.UTF-8이 지원되지 않는다는 등..) 다음을 따름
          locale -a     현재 이용가능한 로케일 표시(확인차)
          sudo vi /etc/locale.gen     으로 파일 열어 ko_KR.UTF-8앞의 주석 표시 삭제
          sudo locale-gen     로케일 갱신
          locale -a     로케일이 업데이트 됐는지 확인

         
          참고: https://wiki.archlinux.org/index.php/Locale_(%ED%95%9C%EA%B5%AD%EC%96%B4)

빌드가 끝나면
현재 폴더(~/ardupilot/ArduCopter)에 ArduCopter.elf 파일 생성됨

홈으로 옮김 (ArduCopter.elf 파일이 옮겨지기 전까지는 얼브레인은 3색 led 모두 켜지는 에러상태임)
mv ArduCopter.elf ~/ArduCopter.elf
