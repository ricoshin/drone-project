vi ~/apm.sh

 $APM_BIN_DIR/ArduCopter.elf -A udp:10.0.0.2:5000 -B /dev/ttyAMA0 -C udp:192.168.0.44:8000 $FLAGS

빨간색 : primary 텔레메트리 -> WiFi hotspot 연결시
파란색 : secondary 텔레메트리 -> 이더넷/WiFi client로 연결시

  1. File탭에서 Advanced Mode 체크
  2. communication 탭 -> Add Link -> UDP 클릭
  3. 5000 또는 8000과 같은 포트 번호 입력하고 connect

*연결 안될 때는 APM planner 재시작
http://forum.erlerobotics.com/t/solved-cant-connect-to-apm-via-wifi/364/6

autopilot은 /home/에 위치하는 apm.sh이라는 스크립트에 의해 launched된다. (반드시 홈에 위치해야함)
apm.sh을 편집함으로써, 어떻게 autopilot을 실행하고 어떤 vehicle의 autopilot을 실행할지 선택할 수 있다.

./<ArduCopter_executable_name.elf> -A <primary_telemetry_port> -B <GPS_port> -C <secondary_telemetry_port>

#Launch ArduCopter.elf excutable, send data through USB connection (192.168.7.1) and attach GPS to port ttyo5./ArduCopter.elf -A udp:192.168.7.1:6000 -B /dev/ttyO5 #Launch hexa frame ArduCopter.elf excutable, send data through WiFi connection (11.0.0.2) and attach GPS to port ttyo5./ArduCopter_hexa.elf -A udp:10.0.0.2:6000 -B /dev/ttyO5 #Launch ArduPlane.elf excutable, send data through MAVROS bridge (127.0.0.1) and attach GPS to port ttyo5./ArduPlane.elf -A udp:127.0.0.1:6001 -B /dev/ttyO5 #Launch Rover autopilot, use USB0 us telemetry port and WiFi as secondary telemtry port#Use GPS located at ttyo5./ArduRover.elf -A /dev/ttyUSB0 -B /dev/ttyO5 -C udp:10.0.0.2:6000

udp:10.0.0.2:6000 WiFi(Hot spot)
udp:192.168.7.1:6000 같은 공유기 밑에 있는 네트워크 컴퓨터
udp:127.0.0.1:6001 MAVROS bridge
/dev/tty05 port ttyo5 (GPS port)
/dev/ttyUSB0 USB0 (915Mhz Radio)
/dev/ttyAMA0 UART 포트
