[Raspberry Pi2] Navtive vs. Cross compile

데스크탑에서 빌드한 실행파일을 라즈베리파이에서 실행하려고 하면 exec format이 적합하지 않다는 에러는 밷음.
실행파일이 플랫폼에 따라 다른 바이너리 형식을 가지기 때문.

예를 들어, uname -a 해보면 알 수 있는데, 데스크탑의 경우 아키텍처가 x86_64, rPi의 경우 armv7l인 것을 확인할 수 있다.


고로 직접 natively 컴파일 하거나 크로스 컴파일러를 이용해서 빌드해야만 함.
라즈베리 상에서 오픈 플랫폼 빌드하면 Pi 1의 경우 한시간도 걸리고 Pi 2는 많이 빨라져서 -j4 플래그 사용해서 15분정도 걸린다고 함.
데스크탑에 비해 느린 것이 당연함.




You use a cross compiler to produce executables (or objects) for a platform other than the local host.
The native compiler only produces native binaries.

크로스 컴파일 : 로컬 호스트와 다른 플랫폼을 위한 실행파일을 빌드. (예. 데스크탑에서 라즈베리 파이를 위한 실행파일 생성)
네이티브 컴파일 : 로컬 호스트에서 자신의 플랫폼을 위한 실행파일을 빌드.



라즈베이파이에서 mavros의 빌드시 다음과 같은 에러 발생
rasp c++: internal compiler error: Killed (program cc1plus)

메모리 문제일 가능성. swap 공간을 확장해주는 솔루션 시도.

dd if=/dev/zero of=/root/swap bs=1M count=512
chmod 600 /root/swap
mkswap /root/swap
swapon /root/swap

The configuration file is:
/etc/dphys-swapfile 

The content is very simple. By default my Raspbian has 100MB of swap:
CONF_SWAPSIZE=350

If you want to change the size, you need to modify the number and restart dphys-swapfile:
/etc/init.d/dphys-swapfile stop
/etc/init.d/dphys-swapfile start
