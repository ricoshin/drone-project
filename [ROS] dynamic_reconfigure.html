[ROS] dynamic_reconfigure

http://wiki.ros.org/dynamic_reconfigure

노드를 재시작하지 않고도 언제든 노드 파라미터를 수정(재설정)할 수 있도록 해주는 패키지.

현재 dynamic_reconfigure의 초점은 노드 파라미터 중 일부를 외부로부터 재설정이 가능하도록 하는 표준 방법을 제공하는 것이다.
클라이언트 프로그램(예. GUI)은 노드에게 재설정 가능한 파라미터 세트와 그들의 이름, 타입, 범위 등에 대한 정보를 요청할 수 있으며,
유저에게 맞춤형 인터페이스로 표현할 수 있다.

reconfigure_gui
- rqt_reconfigure.

dynparam 커맨드라인 툴

dynparam 툴은 커맨드라인에서 노드의 재설정을 가능하게 할 뿐 아니라, 설정을 파일로 저장하거나 불러올 수 있다.
dynparam 실행을 위한 명령어는 다음과 같다 : 

$ rosrun dynamic_reconfigure dynparam COMMAND
현재 지원하는 명령어는 다음과 과 같다. 
*단, 공통 되는 추가 플래그는 -t가 있으며 -t secs와 같이 입력하면 해당 초 만큼의 타임아웃 시간을 설정함.

1.dynparam list : 재설정 가능한 노드 리스트 (알파벳 순서로) *주의 : 재설정 가능한 파라미터를 보여주는 게 아니라 노드를 보여줌
$ rosrun dynamic_reconfigure dynparam list
2.dynparam get : 노드 설정 가져오기
$ rosrun dynamic_reconfigure dynparam get /node
3.dynparam set : 노드 설정
$ rosrun dynamic_reconfigure dynparam set /node parameter_name value # 한 개 값만 수정
$ rosrun dynamic_reconfigure dynparam set wge100_camera "{'camera_url':'foo', 'brightness':58}" # 값 여러개 수정
4.dynparam set_from_parameter : 파라미터 서버로부터 설정 복사해오기
$ rosrun dynamic_reconfigure dynparam set_from_parameters /node
5.dynparam dump : 파일로 설정 내보내기
rosrun dynamic_reconfigure dynparam dump /node dump.yaml
6.dynparam load : 파일로부터 설정 불러오기
rosrun dynamic_reconfigure dynparam load /node dump.yaml

기본 설정

- 패키지를 만들어 보자.
우리가 만들고자 하는 패키지 이름 : dynamic_tutorials
의존 패키지 : rospy, roscpp, dynamic_reconfigure
catkin_create_pkg --rosdistro ROSDISTRO dynamic_tutorials rospy roscpp dynamic_reconfigure
여기서 ROSDISTRO는 현재 사용중인 ROS 배포판이다.

- 이제 패키지 폴더 안에 cfg 디렉토리를 생성한다.
mkdir cfg #msg, srv와 같은 경로에 있음
- 폴더 안에 만든 Tutorials.cfg 파일을 생성한다.

Tutorials.cfg 파일 작성법

   1 #!/usr/bin/env python
   2 PACKAGE = "dynamic_tutorials"
   3 
   4 from dynamic_reconfigure.parameter_generator_catkin import * # ROS 초기화 / 파라미터 생성기 import
   5 
   6 gen = ParameterGenerator() # 파라미터 생성기 인스턴스 얻기. add 함수를 통해 파라미터 리스트 추가 가능.
   7 
     # add 함수 파라미터
     # gen.add(name, type, level, description, default, min, max)
     # name : 파라미터가 저장될 이름 (string)
     # type : 저장될 값의 타입 (int_t, double_t, bool_t)
     # level : 추후 dynamic reconfigure 콜백에 전달될 bitmask. 콜백이 불리면, 모든 바뀐 level값들을 전부 OR연산한 결과를 보냄.
     # description :  파라미터를 설명하는 문자열 (string)
     # default : 기본값 설정대
     # min : 최소값 설정 (선택사항이며, string과 bool타입에는 적용안됨)
     # max : 최대값 설정 (선택사항이며, string과 bool타입에는 적용안됨)

     # 여러 타입의 파라미터를 추가
   8 gen.add("int_param",    int_t,    0, "An Integer parameter", 50,  0, 100)
   9 gen.add("double_param", double_t, 0, "A double parameter",    .5, 0,   1)
  10 gen.add("str_param",    str_t,    0, "A string parameter",  "Hello World")
  11 gen.add("bool_param",   bool_t,   0, "A Boolean parameter",  True) 
  12 

     # 열거형(enum)인 정수타입 파라미터를 정의.
     # (사이즈에 대한 열거형. 작음:0/중간:1/큼:2/겁나큼:3 : 숫자대신 문자열로 표현 가능케함.)
  13 size_enum = gen.enum([ gen.const("Small",      int_t, 0, "A small constant"),
  14                        gen.const("Medium",     int_t, 1, "A medium constant"),
  15                        gen.const("Large",      int_t, 2, "A large constant"),
  16                        gen.const("ExtraLarge", int_t, 3, "An extra large constant")],
  17                      "An enum to set size")
  18 
     # 방금 생성한 열거형 파라미터를 추가
  19 gen.add("size", int_t, 0, "A size parameter which is edited via an enum", 1, 0, 3, edit_method=size_enum)
  20 
     # 파라미터 생성기에게 필요한 파일을 생성하고 프로그램을 종료하라고 요청함.
     # 두번째 인수는 동적 파라미터를 생성할 노드의 이름(문서 생성에만 사용됨).
     # 세번째 인수는 생성할 파일이름 앞에 붙을 접두사.
     # (c++의 경우에는 TutorialsConfig.h를 생성. python의 경우에는 TutorialsConfig.py를 생성.)
     # 주의 : 세번째 인수의 경우 확장자를 제외하고는 cfg파일과 이름이 같아야 함.
     #       그렇지 않으면 빌드때마다 라이브러리가 새로 생성되며, 그 라이브러리를 사용하는 노드 역시 다시 컴파일이 된다.
  21 exit(gen.generate(PACKAGE, "dynamic_tutorials", "Tutorials"))

cfg 파일의 사용

cfg파일을 사용하기 위해서는 실행 가능하도록 권한을 변경해주어야 한다 : 
chmod a+x cfg/Tutorials.cfg

그 다음은 CMakeLists.txt에 다음을 추가해준다 :
# dynamic reconfigure API 추가
# find_package(catkin REQUIRED dynamic_reconfigure)
generate_dynamic_reconfigure_options(
  cfg/Tutorials.cfg
  #...
)

# 노드를 빌드하기 전에 configure 헤더파일을 먼저 빌드할 수 있도록 의존성 조건 추가
add_dependencies(example_node ${PROJECT_NAME}_gencfg)

이제 패키지를 빌드하면 cfg가 실행되도록 설정되었다. 패키지를 빌드하자!

Tutorial : 노드에서 dynamic reconfigure 세팅하기 (cpp로 작성시)

코드

dynamic_tutorials 패키지 경로로 가서 nodes라는 디렉토리를 만들고 그 안에 server.cpp라는 파일을 생성한다.
server.cpp에 다음의 내용을 집어넣는다 : 

줄 번호 보이기/숨기기
   1 #include <ros/ros.h>
   2 
   3 #include <dynamic_reconfigure/server.h>
   4 #include <dynamic_tutorials/TutorialsConfig.h> # 위에 만든 cfg파일에 의해 생성된 헤더
   5 
     # dynamic_reconfiguration에 새로운 설정이 보내질때마다 호출되는 콜백
     # 첫번째 인수 : 전달받은 새로운 config / 두번째 인수 : 변화된 모든 파라미터의 level값을 OR연산한 결과
     # 두번째 인수인 level값으로 뭘 할지는 엿장수 마음임. 지금은 불필요함.
   6 void callback(dynamic_tutorials::TutorialsConfig &config, uint32_t level) {
       # 테스트를 위해 재설정값을 단순히 출력만 함.
   7   ROS_INFO("Reconfigure Request: %d %f %s %s %d", 
   8             config.int_param, config.double_param, 
   9             config.str_param.c_str(), 
  10             config.bool_param?"True":"False", 
  11             config.size);
  12 }
  13 
  14 int main(int argc, char **argv) {
  15   ros::init(argc, argv, "dynamic_tutorials");
  16 
       # 동적 재설정 서버 정의. <config타입> 넘겨줌. 서버가 살아있는 한(이경우에는 메인문이 끝날때까지), 노드는 재설정 요청을 기다림.
  17   dynamic_reconfigure::Server<dynamic_tutorials::TutorialsConfig> server;
       # 재설정 요청이 들어올 때마다 호출할 콜백 변수를 정의
  18   dynamic_reconfigure::Server<dynamic_tutorials::TutorialsConfig>::CallbackType f;
  19 
       # 콜백 변수에 위에 선언한 콜백 함수를 연결해줌
  20   f = boost::bind(&callback, _1, _2);
       # 콜백 변수를 서버의 콜백으로 설정해줌. 이제 서버가 재설정 요청을 받으면 위의 callback함수를 호출함.
  21   server.setCallback(f);
  22 
  23   ROS_INFO("Spinning node");
       # 노드가 돌면서(spin) 계속해서 요청을 기다림
  24   ros::spin();
  25   return 0;
  26 }

실행하기

이제, 노드를 컴파일 한다. (CMakeList.txt에 add_executable()을 추가하는 것을 잊지마라!)
노드를 먼저 실행한 뒤, rqt_reconfigure를 실행한다. 

$ rosrun rqt_reconfigure rqt_reconfigure

드롭다운 메뉴에서 방금 우리가 만든 노드의 이름인 /dynamic_tutorials가 보이면 성공!
이제 이 노드는 재설정 가능한 노드로써 빌드된 것이다.
