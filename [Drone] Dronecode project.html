DroneKit-Python
온보드 companion computer에서 실행되어 ArduPilot flight controller와 low-latency 통신을 하는 어플리케이션을 만드는 개발키트.
뿐만아니라 드론과 high-latency RF통신을 하는 ground station 어플리케이션도 역시 개발 가능.
MAVLink를 통해 드론과 커뮤니케이션하는데, 이는 드론의 텔레메트리, 상태정보, 파라메터 정보에 프로그램적으로 접근할 수 있도록 해주며, 미션 관리나 드론의 움직임 또는 작동을 직접 컨트롤할 수 있게 한다.

companion computer는 드론에 직접 탑재되어 autopilot과 저지연(low-latency) 링크를 통해 통신하는 장비이다.
companion computer에서 실행되는 어플리케이션들은 계산량이 많거나 시간에 민감한 테스크(컴퓨터 비전, 경로 계획, 3차원 모델링)를 수행함으로써
autopilot만 있을때 보다 훨씬 더 강한 지능을 부여해준다.
DroneKit는 여러 종류의 리눅스가 돌아가는 온보드 컴퓨터에서 사용될 수 있다. 
(파이썬과 인터넷으로부터 다운받을수있는 파이썬 패키지를 이용가능한 환경- 라즈베리파이, 인텔 에디슨, 비글본블랙, 오드로이드)


Communication Protocol

MAVLink(Micro Air Vehicle Communication protocol)
MAVLink는 onboard 아키텍처의 중심부가 될 필요가 없다.
제공되는 missionlib이 파라미터와 미션/웨이포인트 전송을 핸들링한다.
autopilot은 단순히 적합한 데이터 구조체로부터 값을 읽어오기만 하면 된다.
MAVLink는 매우 안정적인 메세지 형식을 갖는다.
많은 GCS와 autopilot이 MAVLink를 지원하는 이유가 그것이다.
MAVLink는 header-only 라이브러리로써, MCU에서 컴파일 할 필요가 없이 include만 해주면 사용할 수 있다.
UAVCAN
우주항공/로보틱스 어플리케이션에서의 안정적인 커뮤니케이션을 위해 디자인된 경량 프로토컬. 캔통신 이용.


Flight Code
APM/ArduPilot은 세계를 리드하는 오픈소스 UAV플랫폼이다. 멀티콥터, 고정익 비행체, 전통척 헬리콥터, 틸트로터, 테일 싯터, 보트, 지상용 차량 등 다양한 종류의 vehicle 타입을 제어할 수 있다. (GPLv3 licensed)
DIY Drones community에 의해 2007년에 만들어졌으며 다양한 소프트웨어와 하드웨어를 지원하기 위해 시작되었다.
(APM/ArduPilot과 Pixhawk/PX4 계열의 autopilot, 리눅스 기반의 autopilot 보드, APM:Copter, APM:Plane, APM:Rover, 그리고 SITL 소프트웨어 프로젝트 등)

PX4는 독립적인 오픈소스/오픈하드웨어 프로젝트이다. 학술/취미/산업 커뮤니티에 낮은 가격과 높은 유용성으로  high-end autopilot을 보급하려는 목표로 시작되었다.

Compatible Ground Control Stations

QgroundControl
•Open-source MAVLink Micro Air Vehicle Communication Protocol with lightweight serialization functions for microcontrollers
•Windows / Linux / MacOS support
•2/3D aerial maps (Google Earth support) with drag-and-drop waypoints
•In-flight manipulation of waypoints and onboard parameters (in EEPROM)
•Real-time plotting of sensor and telemetry data
•Logging and plotting of sensor logs
•Support for UDP, serial (radio modem) and mesh networks
•Supports multiple autopilots (pxIMU, APM/ArduPilot, SLUGS, MatrixPilot/UAVDevBoard, many more)
•MAVLink protocol supports up to 255 vehicles in parallel and project-specific custom messages can be added
•Head-up-display, support for digital video transmission/display
•Competition-proven at IMAV and EMAV competitions (QGroundControl was used in the EMAV 2009 1st place Indoor Autonomy entry)

Mission Planner
Mission Planner is a fully featured GCS running on the Windows Operating System.  Its features include:
•Point-and-click waypoint entry, using Google Maps/Bing/Open street maps/Custom WMS.
•Select mission commands from drop-down menus
•Download mission log files and analyze them
•Configure APM settings for your airframe
•Interface with a PC flight simulator to create a full hardware-in-the-loop UAV simulator.
•In flight HUD view

APM Planner2
APM Planner is a multi OS fully featured GCS.  It runs on MAC OS X, Windows and Linux.  Its features include:
•Configure and calibrate your APM or PX4 autopilot for autonomous vehicle control.
•Upload the latest firmware to your autopilot
•Plan a mission with GPS waypoints and control events.
•Connect a 3DR Radio to view live data and initiate commands in flight.
•View vehicle status and flight data using the head-up display (HUD) area of the Flight Data screen.

DroidPlanner 2 & 3 (Tower) - Not for developers, Android only
DroidPlanner 2 is a GCS for the Android OS 4.0 and above.  It has extensive features including:
•Specifically designed for 3DR multirotors and Iris
•New telemetry screen showing quick glanceable info: HUD, battery, RSSI, distance
•Easy to use Home, Land, and Loiter buttons
•New guided mode with changeable altitude
•Quick mode switching
•New planning screen for quick mission generation
•Easy and powerful mission editing tools
•Basic radio TX setup
•Preflight checklist


AndroPilot - Mobile/table users, Android only
AndroPilot is a GCS for the Android OS 4.0 and above.  It has a large number of features including:

•Bluetooth to serial adapters - useful for phones that don't support USB host mode.
•Map view
•Waypoint editing (including setting waypoint type/parameters etc...)

•Vehicle mode display & control (AUTO, STABILIZE, etc...)
•Speech output (for mode, altitude and battery alerts)
•Capture tlog files to sdcard
•Direct connect to USB on the APM (the only Android GCS with this feature)
•RC channel 'raw' view
•3DR telemetry module support
•Warnings for low battery, low rssi, or low GPS sat count
•UDP link (either as a client or a server)
•TCP link (as a client)
•Special support for the Archos GamePad
•Monitor your vehicle through an optional Pebble smart watch.

Supported Simulation Environments

A flexible simulation framework allows the easy addition of new simulation backends.

JMavSim
jMAVSim is a simple and lightweight multirotors simulator. It can work with the autopilot via serial port or UDP connection, directly using the MAVLink protocol, without any wrappers or ground control stations. Optional secondary port (e.g. UDP) can be configured for ground station connection.

SITL
The SITL (software in the loop) simulator allows you to run Plane, Copter or Rover without any hardware. It is a build of the autopilot code using an ordinary C++ compiler, giving you a native executable that allows you to test the behaviour of the code without hardware.

ROS Gazebo Simulator
Gazebo is a PX4 compatible advanced robotics simulation environment that allows you to simulate the complete system to test new controllers or estimators. In addition to simulating avionics, Gazebo provides the capability to replicate cameras and laser sensors.


Other Compatible Simulators:

Last_Letter
last_letter is a collection of ROS packages suitable for Unmanned Aerial Vehicle research, control algorithm development, non-linear physics simulation, Software-in-the-Loop (SIL) simulation and Hardware-in-the-Loop (HIL) simulation. It is developed by George Zogopoulos-Papaliakos. Currently Last_Letter is in the early stages of development and is ssed for fixed wing and tiltrotor simulation in APM/ArduPilot SITL.

JSBSim
JSBSim is an open source software library that models the dynamics of flight of an aerospace vehicle. The library can be incorporated into larger simulation packages (such as FlightGear and OpenEaagles), or it can be called from a small standalone program to create a batch simulation tool. JSBSim has been in development and use since 1996, and has been built on all of the most popular platforms in use today including those running Linux, Macintosh, and Windows operating systems. Used for fixed wing simulation

CRRCSim
CRRCSim is a model-airplane flight simulation program for Linux with ports to other platforms. Used for fixed wing and helicopter simulation in APM/ArduPilot SITL

X-Plane
X-Plane is a accurate flight simulator which supports fixed wing models. Only XPlane 10 is officially supported and tested with PX4 but (untested) legacy support for XPlane 9 is available in QGroundControl.
