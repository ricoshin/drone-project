ardrone_autonomy

ardrone_autonomy는 Parrot AR-Drone 1.0 & 2.0 쿼드로콥터를 위한 ROS 드라이버이다.
공식 AR-Drone SDK 버전 2.0.1에 기반하고 있다. ardrone_autonomy는 AR-Drone Brown 드라이버의 포크버전이다.
이 패키지는 Simon Fraser University의 Autonomy Lab에서 Mani Monajjemi와 다른 컨트리뷰터에 의해 개발되었다.



사용법

드라이버의 실행 노드는 ardone_driver이다.
$ rosrun ardrone_autonomy ardrone_driver를 직접 실행하거나
원하는 파라미터를 설정한 launch 파일로 실행할 수 있다.




드론으로부터 데이터 읽어들이기


업데이트 주기

드론 업데이트 주기 (드론 -> 드라이버 데이터 전송)
navdata_demo parameter에 의해 설정
- 1로 설정하면 15Hz 
- 0으로 설정하면 200Hz (default)

드라이버 업데이트 주기 (드라이버 -> ROS 토픽 발행)
realtime_navdata 파라미터에 의해 설정
- True이면 드라이버가 받은 데이터를 즉각적으로 퍼블리쉬.
- False이면 받은 데이터를 캐시에 먼저 저장했다가 고정된 주기에 맞춰서 퍼블리쉬. 주기는 looprate 파라미터에 의해 설정.
*기본값은 realtime_navdata=False이고 looprate=50
*looprate가 드론의 전송 주파수보다 작으면 데이터가 유실될 수 있음.
*드라이버가 시작할때 현재 설정을 확인가능. rostopic hz 명령어로 드라이버의 퍼블리쉬 속도를 확인가능.

# Default Setting - 50Hz non-realtime update, the drone transmission rate is 200Hz
$ rosrun ardrone_autonomy ardrone_driver _realtime_navdata:=False  _navdata_demo:=0

# 200Hz real-time update
$ rosrun ardrone_autonomy ardrone_driver _realtime_navdata:=True _navdata_demo:=0

# 15Hz real-rime update
$ rosrun ardrone_autonomy ardrone_driver _realtime_navdata:=True _navdata_demo:=1

IMU data (1.4 버전 리뉴얼)

선가속도, 각속도, 드론의 방향은 ROS 표준인 sensor_msgs/Imu 메세지를 통해 퍼블리쉬 된다.
모든 단위는 미터법을 따르며 TF 기준 프레임은 드론의 베이스 프레임으로 세팅된다.
공분산 값은 cov/imu_la, cov/imu_av, cov/imu_or 파라미터를 통해 확인할 수 있다.

*tf는 유저가 여러개의 좌표 프레임을 지속적으로 추적하도록 해주는 ROS 패키지이다.
tf는 좌표계 사이의 관계를 시간이 지나도 계속 저장하고 있다가
어느 시점에서든 유저가 점이나 벡터 등을 두 좌표계 사이에서 변환할 수 있도록 해준다.


선택적 Navdata (발전된 형태)

Selective Navdata를 이용하여 AR-Drone의 거의 모든 센서값과 디버그값, 상태 리포트 등에 접근할 수 있다.
아래 파라미터들 중 어떤 하나를 True로 만들어주면 해당되는 Navdata 정보가 개별적 토픽으로 발행된다. (기본값은 모두 false)
예) enable_navdata_time = true로 만들어주면 ardrone/navdata_time으로 AR-Drone의 시간 정보가 발행.

카메라

2개의 카메라 사용가능 (전방 카메라, 하방 카메라)
드라이버는 총 세개의 카메라관련 토픽을 생성한다 :
- ardrone/image_raw
- ardrone/front/image_raw
- ardrone/bottom/image_raw

이들 모두는 표준 ROS 카메라 드라이버의 인터페이스이며  image transport 타입의 메세지를 발행한다.
표준 카메라 드라이버를 사용하므로 캘리브레이션 정보는 camera_info 토픽을 통해 발행된다.
캘리브레이션 정보는 ROS 파라미터들에 의해 제공되거나 ardrone_front.yaml과 ardrone_bottom.yaml 파일을 통해서 제공될 수 있다.

AR-Drone 1은 네가지 비디오 스트림 모드를 지원한다.
전방 카메라 이미지 / 하방 카메라 이미지 / 전방 카메라 이미지 내부에 하방 카메라 이미지 표시 / 하방 카메라 이미지 내부에 전방 카메라 이미지 표시
활성화된 설정 모드에 따라서, 드라이버가 PIP(Picture in Picture) 스트림을 분해하고 해당 토픽에 순수한 전방 또는 하방 이미지 스트림을 발행한다.

AR-Drone 2는 PIP 기능을 더이상 지원하지 않는다. 그러므로 어떤 카메라를 선택했냐에 따라서 둘 중 하나의 토픽만 업데이트 된다.



태그 디텍션

Navdata 메세지는 드론의 온보드 영상처리 시스템에 의해 감지된 특별한 태그에 대한 정보도 함께 포함한다.
두 카메라 모두 30프레임의 속도로 태그를 디텍트할 수 있다. 이 기능은 파라미터에 의해 키거나 끌 수 있다.
이 태그에 대한 정보는 레거시 네비게이션 데이터 메세지 필드를 통해 발행된다.


좌표 프레임

좌표 프레임 : odom, ${base_prefix}_link, ${base_prefix}_frontcam, ${tf_prefix}/${base_prefix}_bottomcam
${base_link}는 세 개의 좌표계가 공유하는 접두어이며, 파라미터를 이용하여 세팅할 수 있다. 기본 값은 ardrone_base이다.

드라이버는 이 4개의 좌표계들 사이에 3개의 TF 트랜스폼을 발행한다. 

설정된 프레임 이름은 발행되는 모든 토픽(navdata, imu, camera)의 헤더 안에 있는 frame_id에 저장된다.
모든 프레임은  ROS REP 103 호환 (ROS에서 사용되는 단위와 좌표계 컨벤션에 대한 레퍼런스 / REP : ROS Enhancement Proposal)




AR-Drone으로 명령 보내기



다음은 해당 토픽에 대하여 ROS 표준 메시지인 std_msgs/Empty를 발행한 결과를 나타낸다.
- ardrone/takeoff --> 드론 이륙
- ardrone/land --> 드론 착륙
- ardrone/reset --> 비상정지/리셋

이륙 후에 비행을 위해서는 cmd_vel 토픽에 geometry_msgs::Twist타입의 메세지를 발행하면 된다.

# geometry_msgs/Twist.msg
# 자유 공간에서의 속도를 선속도와 각속도로 나누어 표현 (Vector3는 x,y,z를 멤버변수로 갖는다)
Vector3  linear
Vector3  angular

-linear.x: move backward
+linear.x: move forward
-linear.y: move right
+linear.y: move left
-linear.z: move down
+linear.z: move up

-angular.z: turn right
+angular.z: turn left
각 요소의 범위는 -1.0에서 1.0 사이여야 한다. 최대 범위는 파라미터를 사용하여 설정할 수 있다.


호버 모드

위에서 사용하고 있지않는 다른 두개의 멤버변수인 angular.x와 angular.y를 사용하여 "auto-hover"모드를 키고 끌 수 있다.
모든 6개의 요소가 0으로 설정되면 "auto-hover"가 작동된다.
혹시라도 작동을 원하지 않으면 위의 4개 변수는 0으로 설정하고 나머지 두 개는 영이 아닌 아무 값으로 채우면 된다.

카메라 토글

아무 파라미터 없이 ardrone/togglecam 서비스를 호출하면 활성화 카메라 영상 스트림을 바꿀 수 있다. (전방 카메라 <--> 후방카메라)
명령어는 다음과 같다 : rosservice call /ardrone/togglecam

ardrone/setcamchannel 서비스는 현재 환성화할 채널을 직접 바꿀 수 있도록 해준다.
'unit8 channel'타입의 파라미터(바꿀 채널) 한개를 호출시 반드시 지정해주어야 함 :
AR-Drone 1.0의 경우 {0,1,2,3} / AR-Drone 2.0의 경우 {0,1} 값을 가질 수 있으며 값이 의미하는 모드는 카메라 섹션에서 설명하고 있는 순서와 같음.


LED 애니메이션

ardrone/setledanimation 서비스는 14개의 미리 정의된 LED 애니메이션을 호출한다.
파라메터는 다음과 같다
 - uint8 type : 0~13까지의 숫자 범위를 갖는 애니메이션 타입
 - float32 freq : 애니메이션의 주파수 (Hz)
 - uint8 duration : 애니메이션의 지속시간 (초)

type 파라미터는 다음과 같은 순서로 0부터 13까지의 숫자로 매핑된다.
BLINK_GREEN_RED, BLINK_GREEN, BLINK_RED, BLINK_ORANGE,
SNAKE_GREEN_RED, FIRE, STANDARD, RED, GREEN, RED_SNAKE,BLANK,
LEFT_GREEN_RIGHT_RED, LEFT_RED_RIGHT_GREEN, BLINK_STANDARD
다음과 같은 명령어로 테스트해볼 수 있다.
$ rosservice call /ardrone/setledanimation 1 4 5


비행 애니메이션

ardrone/setflightanimation 서비스를 호출하면 20개의 미리 정의된 비행 애니매이션을 실행한다.
파라미터는 다음과 같다
- uint8 type : 0~19까지의 숫자 범위를 갖는 비행 애니메이션 타입
- unint16 duration : 애니메이션의 지속시간 기본값인 0을 사용하는 것을 추천

type 파라미터는 다음과 같은 순서로 0부터 19까지의 숫자로 매핑된다.
ARDRONE_ANIM_PHI_M30_DEG, ARDRONE_ANIM_PHI_30_DEG, ARDRONE_ANIM_THETA_M30_DEG, ARDRONE_ANIM_THETA_30_DEG,

ARDRONE_ANIM_THETA_DANCE, ARDRONE_ANIM_VZ_DANCE, ARDRONE_ANIM_WAVE, ARDRONE_ANIM_PHI_THETA_MIXED,
ARDRONE_ANIM_DOUBLE_PHI_THETA_MIXED, ARDRONE_ANIM_FLIP_AHEAD, ARDRONE_ANIM_FLIP_BEHIND, ARDRONE_ANIM_FLIP_LEFT,
ARDRONE_ANIM_FLIP_RIGHT

드론이 비행 중일 때 다음과 같은 명령어로 테스트해볼 수 있다.
rosservice call /ardrone/setflightanimation 1 0

Flat Trim

아무 파라미터 없이 ardrone/flattrim 서비스를 호출하면 AR-Drone에 "Flat Trim" 요청을 보냄으로써 드론이 평평한 표면에 있다고 가정하고 회전 예측량을 다시 캘리브레이션 한다. 드론이 비행중이거나 실제로 평평한 표면에 있지 않을 때는 호출하지 않는다.


USB 스틱에 저장

ardonre/setrecord 서비스를 호출하면 USB 스틱에 저장하는 것을 작동시키거나(1을 보냄) 멈출(0을 보냄) 수 있다.

가속도 센서 ● acceleration sensor 기체가 수평 보조장치로써 자이로와 함께 사용되며 중력가속도를 바탕으로 각도를 측정하는 장치
자이로(gyro) ● 기체가 수평을 잡도록 도와주는 센서장치. 10만원 전후의 연습용 수준의 드론에도 장착되어 있을 정도로, 안전한 비행을 돕는 기초적인 센서이다.
