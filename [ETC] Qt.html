Qt는 무엇인가?
Qt는 데스크탑, 임베디드, 모바일을 위한 크로스 플랫폼 어플리케이션 디벨럽먼트 프레임 워크 이다
지원되는 플랫폼은  Linux, OS X, Windows, VxWorks, QNX, Android, iOS, BlackBerry, Sailfish OS 등을 포함한다.
Qt는 자신에 의한 프로그래밍 언어는 없습니다. 이것은 C ++로 작성된 프레임 워크입니다. 
전처리는 MOC (메타 오브젝트 컴파일러)는 시그널과 슬롯 등의 기능을 갖춘 C ++ 언어를 확장하는 데 사용됩니다.
컴파일 단계 전에 MOC는 + Qt의 확장 C 언어로 작성된 소스 파일을 분석하고 거기에서 표준 기반의 C ++ 소스를 생성합니다. 따라서 프레임 워크 자체와 그것을 사용하는 응용 프로그램 / 라이브러리 클랜, GCC, ICC, MinGW와 MSVC와 같은 표준 기반의 C ++ 컴파일러로 컴파일 할 수 있습니다.
 
빌드 시스템
어떤 빌드 시스템은 Qt에서 사용할 수 있지만, Qt는 자신의 qmake를 제공합니다. GNU는 Visual Studio와 Xcode를 만들기 위해 그것은 특정 플랫폼 빌드 시스템을위한 크로스 플랫폼의 프론트 엔드입니다.
CMake 도 Qt 프로젝트를 구축하기위한 일반적인 대체이며, Qt 4 지원은 2005 년 이전에 통합되어 있으면 Qt 5 초기 지원을 제공했습니다.
새로운 선수는 최근 게임에 들어간 : Qt는 QBS a.k.a 스위트를 빌드합니다. QBS는 또한 Javascript를 지원 QML 기반 빌드 시스템입니다. 이 빌드 시스템은 능력을 구축 할뿐만 아니라 cmake처럼 포장 제공 할뿐만 아니라.

IDE (Integrated Development Environment : 통합 개발 환경)
Qt는 Qt Creator라는 고유의 통합 개발 환경 (IDE)이 포함되어 있습니다.
이것은 Linux OS X와 Windows에서 실행하고 모든 주요 버전 관리 시스템을 위한 지능형 코드 완성, 구문 강조, 통합 도움말 시스템, 디버거 와 프로파일 러의 통합을 제공합니다. (예를 들어 git, Bazaar)

Widgets
Qt를 통해 GUI를 해당 위젯 모듈을 사용하는 C ++로 직접 기록 할 수있다. Qt는 또한 위젯 기반 GUI 코드 생성기 역할 Qt 디자이너라는 대화 형 그래픽 도구가 포함되어 있습니다. Qt 디자이너는 독립적으로 사용할 수있을뿐만 아니라 Qt Creator에 통합되어 있습니다.  

More Modules
Qt는 GUI 툴킷보다 훨씬 더 많은 것이다. 그것은 네트워킹, 데이터베이스, OpenGL을, 웹 기술, 센서 분야에서 크로스 플랫폼 개발을위한 모듈, 통신 프로토콜 (블루투스, 시리얼 포트, NFC), XML 및 JSON 처리, 인쇄, PDF 생성, 등을 제공합니다.
공식 Qt 모듈의 전체 목록은 Qt는 설명서의 모든 모듈 페이지를 참조하십시오.
또한, Qt 프로젝트에 사용하기 위해 Qt 기반으로 타사 모듈의 목록에 대한 Inqlude.org를 참조하십시오.

Language Bindings
Qt를 사용하는 응용 프로그램은 일반적으로 C ++ 및 QML 바인딩에 기록되어 있지만 다른 언어가 존재합니다. 이들은 Qt의 일부는 아니지만 다양한 타사에서 제공되고 있습니다.
예를 들어 Riverbank Computing은 상용 및 무료 소프트웨어 파이썬 바인딩 (PyQt)를 제공합니다.
언어 바인딩 카테고리를 참조하십시오.
