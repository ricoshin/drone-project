Not recommended to use in automatic control because lack of safety mechanisms.
Use one of setpoint plugins and OFFBOARD mode.

# RC_CHANNELS_OVERRIDE ( #70 )
# RC채널의 RAW값은 MAV(드론)로 보내져서 RC radio로부터 받은 정보를 오버라이드하는데 사용된다.

# uint8_t target_system : System ID
# uint8_t component : Component ID
# uint16_t chan1_raw ... chan8_raw : RC 채널값, in microseconds. UINT16_MAX값을 주면 이 필드를 무시한다.

/mavros/setpoint_attitude/cmd_vel          # (geometry_msgs/TwistStamped) Send angular velocity.
# SET_ATTITUDE_TARGET ( #82 )
# 원하는 기체의 애티튜드(자세, 포즈)를 설정한다. 기체를 조종하기 위한 외부 컨트롤러에 의해 사용된다. (수동 컨트롤 또는 다른 시스템)
# uint32_t time_boot_ms : 시스템 부팅 이후를 기점으로 밀리초로 기록한 타임스템프.
# uint8_t target_system : 시스템 ID
# uint8_t target_component : 컴포넌트 ID
# uint8_t type_mask : 매핑 : 만약 이 비트들이 셋되면, 해당되는 입력은 무시된다.
#                                 (bit 1 : 롤 속도, bit 2 : 피치 속도, bit 3 : 요 속도, bit 4-6 : 예비용,  bit 7 : 쓰로틀, bit 8 : 고도)
# float[4] q : 애티튜드 쿼터니언(사원수). w, x, y, z 순서이며, zero-rotation 값은 1, 0, 0, 0임.
# float body_roll_rate : 초당 라디안 각속도 (롤)
# float body_pitch_rate : 초당 라디안 각속도 (피치)
# float body_yaw_rate : 초당 라디안 각속도 (요)
# float thrust : 0부터 1사이로 정규화된 전체 추력.  (만약 반대방향 추력 발생이 가능한 기체의 경우 -1부터 1 범위)

# Sent position setpoint using SET_POSITION_TARGET_LOCAL_NED.
/mavros/setpoint_position/local               # (geometry_msgs/PoseStamped) Setpoint position. Only YAW axis extracted from orientation field.
# SET_POSITION_TARGET_LOCAL_NED ( #84 )
# 지역적 NED 좌표 프레임 안에서 원하는 기체의 위치를 설정한다. 기체를 조종하기 위한 외부 컨트롤러에 의해 사용된다. (수동 컨트롤로 또는 다른 시스템)
# uint32_t time_boot_ms : 시스템 부팅 이후를 기점으로 밀리초로 기록한 타임스템프.
# uint8_t target_system : 시스템 ID
# uint8_t target_component : 컴포넌트 ID
# uint8_t coordinate_frame : 선택할 수 있는 옵션들  (MAV_FRAME_LOCAL_NED = 1, MAV_FRAME_LOCAL_OFFSET_NED = 7,
                                                                                                MAV_FRAME_BODY_NED = 8, MAV_FRAME_BODY_OFFSET_NED = 9)
# uint8_t type_mask : 매핑 : 만약 이 비트들이 셋되면, 해당되는 입력은 무시된다. 10번째 비트를 셋하면 가속도 값 대신에 force 값으로 해석.
#                         ( bit 1: x, bit 2: y, bit 3: z, bit 4: vx, bit 5: vy, bit 6: vz, bit 7: ax, bit 8: ay, bit 9: az, bit 10: is force setpoint, bit 11: yaw, bit 12: yaw rate)
# float x, y, z : NED 프레임에서의 x, y, z좌표 (m)
# float vx, vy, vz : NED 프레임에서의 x, y, z 방향 속도 (m/s)
# float afx, afy, afz : NED 프레임에서의 x, y, z 방향 가속도(m/s^2) 또는 힘(N)
# float yaw : yaw 설정값 (rad)
# float yaw_rate : yaw 각속도 설정값 (rad/s)
# GUIDED 모드에서만 작동 : 결국 GPS 락킹에 의존하는 모드로써, 결국 로컬 포지션이라고 결국 상대위치로 이동하기 위하여 GPS 신호를 사용하므로 GPS없이는 무용지물.

GCS를 직접 만들거나 DroneKit로 Companion Computer 앱을 만들때 유용하다.
따로 명시되어 있지 않는한 대부분은 다른 모드에서도 호출 가능하다.

Movement commands

이 커맨드는 GUIDED 모드에서만 부를 수 있다. 위치와 속도로 기체를 컨트롤하기 위해 사용된다!
SET_POSITION_TARGET_LOCAL_NED
SET_POSITION_TARGET_GLOBAL_INT


안녕, position/local 토픽에 대해 질문이 있어. pixhawk에서 px4 펌웨어를 사용중이야.
내가 토픽을 echo하면 Z포지션만 나와. X와 Y포지션은 아무리 움직여도 0.0이야.
난 이게 IMU 기반으로 값이 나올 거라고 기대했는데 말이야.

position/vision토픽에 발행도 해봤지만, 여전히 position/local토픽에는 영향이 없었어.
내가 X,Y값을 받기 위해서 놓친 부분이 있을까? 또 비젼 예측 포즈를 설정하는 부분에는? 
(CBRK_NO_VISION은 0으로 설정했음)

======================

IMU 데이터만으로는 로컬 포지션이 절대 안 바뀔거야. 자연스러워. 포지션 소스가 없으면 예측도 없어.
Z는 기압계 때문에 예측될 수 있는거야. CBRK_NO_VISIOM이 셋 되어있다면 문제가 안될거야. 
포즈 소스를 발행할때 얼마나 빠른 속도였어? 시스템에 시간제한이 있거든.

=========================
하지만 내가 위치에 대한 비전 예측을 입력하면, 로컬 포지션이 나와야하는거 아니야? 비젼 예측 포즈는 10Hz로 발행됐어.
시스템 시간제한이 얼마나 되길래??

==========================
예측이 적절하게 넘어간다면 그래야지. 시간제한은 0.5초 IIRC야. 그 정도면 괜찮은데.
예측값은 어디서 받고 있는거야? inav 메세지에 있는 비전으로부터 첫번째 유효한 예측을 받아서 넘겨야해.

==============================
비젼 예측을 위해 primesense를 사용할거지만, 아직은 없어. 그래서 수동으로 포즈를 보내고 있어,
하지만 내가 보낸 시간이 로컬 포지션 토픽에서의 시간과는 좀 차이가 있을거라 오차가 있긴 해.
